Decisions, Mistakes & Impact

Hackathon to production

Incubated the concept in a hackathon, built POC in 1 month. Initially resource-constrained, spent too much time on implementation details before setting up swimlanes.

Concurrent edit data loss

Original section layout schema caused data loss: removing a column while another user adds content drops their work. Redesigned to a fixed 3-section model with visibility metadata — sections are hidden, never removed. Also introduced constraint-based conflict resolution (First Writer Wins) for concurrent control edits.

Unblocking partner teams

CoAuth rollout blocked by reliability issues from ODSP Push and OpStream teams whose fixes weren’t on their near-term roadmap. Drafted document outlining issue, user impact, and solutions. Offered direct team contribution. Unblocked within a week.

Mistake: late delegation

Held onto too much implementation. At peak, team grew to 7 engineers (mix of senior/junior). Once I broke work into swimlanes — seniors on service integration (OpStream, Push), juniors on client features (presence, exclusive sessions, undo/redo, data loss dashboard) — things accelerated.

What I'd do differently

Set up clear swimlanes from the start. Focus on high-level architecture, let others drive implementation. Would have scaled me faster and grown the team at the same time.

Lesson

Quality and trust outweigh speed. Every data loss incident reinforced that. Delegation isn’t just about reducing load — it’s about enabling the team to scale faster and giving others growth opportunities.

The 3-section model is a great concrete example: original design removed sections on layout change, causing data loss with concurrent edits. Fix: always keep 3 sections, toggle visibility. Constraint-based FWW prevents concurrent control edit conflicts. Be honest about delegation mistake. Mention exclusive session design for publish/schedule (presence-based freeze).

AI Authoring

Decisions, Challenges & Growth

Content-first vs. tool-oriented

Initial plan was tool-oriented orchestration via agent platform. Partner team was developing content-first approach. Built a POC using content-first, achieved MVP without building custom tools. Concrete demo convinced management to pivot.

Smart Sections prototype

Short timeline to validate AI value to customers. Built on two tracks: improved infrastructure + LLM integration. Partnered with senior engineer for vertical slice demo. Infrastructure challenge: existing system couldn’t generate SharePoint web parts. Delivered prototype, built confidence across leadership.

Rapid AI learning

Dove into AI frontier through parallel channels: external research & open papers, internal Microsoft Copilot architectures, hands-on AI tools in own workflow. Ramped within months.

Ambiguity in Vibe Authoring

Project began with unclear goal — “like Word” or “like GitHub Copilot”? No clear boundaries, metrics, or architecture. Defined content-oriented architecture, Git-backed workspaces, and evaluation surfaces to bring clarity.

AI Properties impact

Org-wide schema-driven platform. Reduced token usage by 30%, improved generation quality/performance, reduced model confusion. Became the foundation that unblocked multiple teams.

Evolution from CoAuth

CoAuth taught me to delegate and prioritize reliability. AI Authoring taught me to navigate ambiguity and move fast. Applied swimlane lesson from day one. Together they show I can operate in both multi-year infrastructure and fast-cycle AI spaces.
